    Comprehensive Testing in Bijux CLI¶

    Predictable quality at scale. Bijux CLI ships with a deep, layered test suite (≈ 2,664 tests) covering units, integrations, functional scenarios, and full E2E flows. CI enforces green builds across Python 3.11+ with strict linting, typing, and security gates.

    Testing Philosophy¶

    We optimize for predictability, depth, and maintainability:

    • Determinism — stable outputs, structured errors, fixed flag precedence.
    • Layered depth — unit > integration > functional > E2E, each with clear responsibility.
    • Design pressure — tests encourage separation of concerns (DI-friendly services, pure serialization, thin CLIs).
    • Speed where it matters — fast unit feedback; heavier scenarios gated under markers.

    Back to top

    Suite Layout¶

    Top-level structure under tests/ (abridged):

    tests/
  unit/
    commands/        # audit, config, dev, docs, doctor, help, history, memory, repl, sleep, status, utilities, version
    core/            # context, di, engine, enums, exceptions
    infra/           # emitter, observability, process, retry, serializer, telemetry
    root/            # __init__, __main__, api, cli, httpapi
    services/        # audit, config, docs, doctor, history, memory, plugins, utils
  integration/
  functional/
  e2e/
    api/ dev/ history/ plugins/ repl/ …  # + fixtures

    Shared fixtures live in conftest.py files at appropriate levels; E2E adds JSON/YAML shape fixtures for contract-like assertions.

    Test Types¶

    Unit¶

    Goal: Isolate behavior of a single module/service. Coverage highlights:

    • Core: DI resolution/graphs, engine flow, exceptions/enums consistency.
    • Infra: serializer (JSON/YAML, redaction), retry/backoff w/ jitter & timeouts, observability/telemetry hooks, process management.
    • Services: config/history/memory/plugins/docs/doctor/audit utils.
    • Commands: validation, parsing, structured outputs.

    Depth patterns: parametrization; mocks for file I/O, telemetry, and timing; assertions on side-effects (e.g., history atomic writes, locks).

    Back to top

    Integration¶

    Goal: Verify wiring (service ↔ contract ↔ infra) through DI and orchestrations. Focus: realistic pipelines without the CLI wrapper; concurrency on shared resources (e.g., history locks); config propagation across layers.

    Back to top

    Functional¶

    Goal: Assert user-visible behavior of commands. Focus: global flag precedence (help/quiet/debug/format/pretty/verbose), structured stdout/stderr, exit codes, non-interactive REPL.

    Back to top

    End-to-End (E2E)¶

    Goal: Exercise the installed CLI the way a user would. Scope: plugin lifecycle (scaffold/install/list/info/check/uninstall), REPL sessions, dev tools (DI graphs), API endpoints, performance paths (history). Technique: subprocess runs, fixture-driven shape checks (JSON/YAML), perf thresholds where relevant.

    Back to top

    Fixtures & Data¶

    • Global conftest.py manages temp dirs, env overrides, and deterministic clocks where needed.
    • E2E shape fixtures (e.g., di_shape.json, history_shape.yaml) guard contract stability.
    • Redaction helpers ensure secrets never leak in snapshot/shape comparisons.

    Back to top

    Conventions & Markers¶

    Naming: test_*.py, functions start with test_. Style: pytest idioms—parametrize for matrix cases, minimal mocks, explicit side-effects. Markers:

    • @pytest.mark.slow — performance or network-heavy paths
    • @pytest.mark.e2e — full CLI/subprocess flows
    • @pytest.mark.asyncio — async units/integrations

    Keep tests hermetic: no external network unless explicitly marked, no reliance on machine state.

    Running Tests¶

    From a clean venv:

    # Everything (fastest path via Makefile orchestration)
make test

# Plain pytest
pytest -q

# Tox (multi-Python; mirrors CI)
tox -q -p auto           # or: tox -e py311,py312,py313

    Common extras:

    # With coverage report
pytest --cov=bijux_cli --cov-report=term-missing

# Stop on first failure, short tracebacks
pytest -x --tb=short

    Selecting/Scoping Tests¶

    # Only unit
pytest tests/unit -q

# Only E2E
pytest -m e2e -q

# Exclude slow
pytest -m "not slow" -q

# Keyword selection
pytest -k "plugins and not uninstall" -q

    Coverage & Quality Gates¶

    • Coverage target: ≥ 90% overall (critical modules higher).
    • Docs & style: ruff (lint+format), pydocstyle, interrogate.
    • Typing: mypy, pyright, optional pytype passes for regressions.
    • Security & hygiene: bandit, pip-audit, reuse, codespell, deptry, radon.

    Back to top

    CI Pipeline¶

    GitHub Actions runs on pushes/PRs:

    • Matrix: Python 3.11–3.13; Node 20 + Java 17 where API/Redoc/OpenAPI tooling is used.
    • Stages: lint → type → unit/integration/functional → E2E → docs build → security.
    • Gates: any failure blocks merge; artifacts include coverage and logs.

    Back to top

    Mutation & Property-Based Testing¶

    • Mutation: Cosmic Ray & mutmut validate assertion strength on critical paths (serializer, retry/backoff, config).

    make mutation   # or: cosmic-ray run config/cosmic-ray.toml
    * Property-based: Hypothesis explores edge cases (e.g., serializer inputs, option matrices) beyond example-based tests.

    Back to top

    Performance, Flakes, Determinism¶

    • Mark perf-sensitive tests as slow; assert upper bounds where meaningful.
    • Avoid sleeps; mock clocks/backoffs (retry tests), or patch asyncio.sleep.
    • No reliance on wall-clock randomness; seed where PRNG used.
    • Flakes are failures—quarantine with markers only until fixed.

    Back to top

    Contributing New Tests¶

    • Pick the right layer (unit vs integration vs E2E).
    • Prefer parametrization over similar copy-pasted tests.
    • Assert shapes & side-effects (stdout/stderr, files, telemetry).
    • Keep fixtures local unless they’re broadly reusable.
    • Don’t hit the network or real home directories—use tmp paths and env overrides.
    • Add docs/examples if behavior is user-visible.

    Quick Commands Cheat Sheet¶

    # Fast local loop
pytest tests/unit -q

# Functional + E2E only
pytest -m "functional or e2e" -q

# Coverage HTML
pytest --cov=bijux_cli --cov-report=html && open htmlcov/index.html

# Tox across Pythons
tox -q -p auto

# Lint+type before PR
make lint quality
